# Usability Study Plan - Football Scout Tool

**Project:** DS209 Final Project - Football Scout Tool
**Researcher:** Karim Mattar
**Live Site:** https://karimmattar1.github.io/ds209-final-project/06_Website/
**Study Duration:** 30 minutes per participant
**Minimum Participants:** 3

---

## Study Objectives

Test the Football Scout Tool to identify:
1. Usability issues that prevent task completion
2. Confusing interface elements or navigation
3. Features that work well and should be retained
4. Missing features or information users expect
5. Overall user satisfaction and engagement

---

## Session Script

### Introduction (2 minutes)

**Say this to participants:**

> "Thank you for participating in this usability study. I'm testing a football player analysis tool I built for my Data Visualization class at UC Berkeley.
>
> This session will take about 30 minutes. I'll ask you to complete some tasks while I observe. Please think aloud as you work - tell me what you're thinking, what you're looking for, and if anything confuses you.
>
> Remember: I'm testing the website, not you. There are no wrong answers. If something doesn't work, that's valuable feedback for me. Feel free to be honest about what you like and don't like.
>
> I'll be recording your screen and audio for my analysis. Is that okay with you?"

**Wait for consent, then start recording.**

> "Great! I'm going to share the website link with you now: https://karimmattar1.github.io/ds209-final-project/06_Website/
>
> Please open it and take a moment to look at the homepage. Don't click anything yet - just tell me your first impressions."

**Note their initial reaction (confused? excited? understanding the purpose?)**

## Task List (20 minutes)

### Task 1: Explore the Homepage (2 minutes)

**Instructions to participant:**

> "Take a look at the homepage. What do you think this tool does? Who do you think it's for?"

**What to observe:**
- Do they understand it's about football (soccer), not fantasy football?
- Can they identify the main features?
- Do they understand what data is available?

**Success criteria:**
- [ ] User correctly identifies tool purpose
- [ ] User understands it's real football (soccer) player data
- [ ] User can name at least 2 of the 4 main features

---

### Task 2: Find Top Performers (3 minutes)

**Instructions to participant:**

> "I want to see which forwards have scored the most goals per 90 minutes this season. Can you find this information using the tool?"

**What to observe:**
- Do they know where to look? (Analysis page)
- Can they navigate to the correct page?
- Can they interpret the visualization?
- Can they hover/interact with the chart?

**Success criteria:**
- [ ] User navigates to Analysis page
- [ ] User identifies the "Top Forwards" chart
- [ ] User can name at least 2 top scorers
- [ ] User interacts with tooltip/chart

**If stuck after 1 minute, hint:** *"Try looking in the navigation menu for analysis or statistics."*

---

### Task 3: Compare Two Players (4 minutes)

**Instructions to participant:**

> "I want to compare Mohamed Salah with any other Premier League winger of your choice. Can you do that using this tool?"

**What to observe:**
- Do they find the "Compare Players" page?
- Can they search for player names?
- Is the search/dropdown intuitive?
- Can they interpret the comparison chart?
- Do they understand what the metrics mean?

**Success criteria:**
- [ ] User navigates to Compare page
- [ ] User successfully searches for Salah
- [ ] User selects a second player
- [ ] User views comparison chart
- [ ] User can describe at least one difference between players

**If stuck after 1.5 minutes, hint:** *"Check the navigation menu at the top."*

---

### Task 4: Explore Relationship Between Metrics (5 minutes)

**Instructions to participant:**

> "I'm curious if there's a relationship between how much a player runs with the ball (progressive carries) and how many goals they create. Can you explore this using the tool?"

**What to observe:**
- Do they find the Explorer page?
- Can they understand the scatter plot?
- Can they use filters (position, league)?
- Do they change X/Y axis metrics?
- Can they interpret the visualization?

**Success criteria:**
- [ ] User navigates to Explore page
- [ ] User identifies scatter plot showing relationship
- [ ] User successfully uses at least one filter
- [ ] User can describe the relationship they see

**If stuck after 2 minutes, hint:** *"Try the 'Explore' page in the navigation."*

---

### Task 5: Find Similar Players (4 minutes)

**Instructions to participant:**

> "Let's say Erling Haaland gets injured and Manchester City needs to find a replacement. Can you use this tool to find players with a similar statistical profile?"

**What to observe:**
- Do they find the "Similar Players" page?
- Can they search for Haaland?
- Can they understand similarity scores?
- Do they find the results useful?
- Can they identify potential replacements?

**Success criteria:**
- [ ] User navigates to Similar Players page
- [ ] User searches for Haaland (or similar top striker)
- [ ] User views similar players chart
- [ ] User can name at least 2 similar players
- [ ] User understands what "similarity" means in this context

**If stuck after 1.5 minutes, hint:** *"Look for a feature that helps find similar or comparable players."*

---

### Task 6: Filter by Specific Criteria (2 minutes)

**Instructions to participant:**

> "Show me all midfielders under age 23 from La Liga who have high progressive carries."

**What to observe:**
- Can they navigate back to Explorer?
- Can they use multiple filters simultaneously?
- Do they understand how to filter by position, age, and league?
- Is the filtering intuitive?

**Success criteria:**
- [ ] User navigates to Explore page
- [ ] User applies position filter (MF)
- [ ] User applies league filter (La Liga)
- [ ] User identifies young players with high progressive carries

## Debriefing Questions (5 minutes)

After completing tasks, ask these questions:

### Overall Experience

1. **"On a scale of 1-10, how easy was it to use this tool? Why?"**
   - Note: Rating and reasoning

2. **"What was the most confusing or frustrating part of using this tool?"**
   - Note: Specific pain points

3. **"What did you like most about the tool?"**
   - Note: Positive feedback

### Specific Features

4. **"Did you understand what all the metrics meant (like xG, progressive carries, etc.)?"**
   - Note: Which metrics were unclear

5. **"Was the navigation clear? Could you easily find what you were looking for?"**
   - Note: Navigation issues

6. **"Did the visualizations help you understand the data? Were they clear?"**
   - Note: Visualization clarity

### Missing Features

7. **"Was there anything you expected to see but couldn't find?"**
   - Note: Missing features

8. **"If you could change one thing about this tool, what would it be?"**
   - Note: Top priority change

### Context

9. **"How interested are you in football/soccer? (Very/Somewhat/Not at all)"**
   - Note: User background context

10. **"Would you use this tool again? Why or why not?"**
    - Note: Likelihood of reuse

## Post-Session (Within 24 Hours)

### Review Recording

Watch the recording and note:

1. **Errors made:**
   - Where did the user click the wrong thing?
   - What did they miss?
   - What caused confusion?

2. **Hesitations:**
   - Where did they pause?
   - What made them uncertain?
   - What did they have to think about?

3. **Verbal feedback:**
   - What did they say during think-aloud?
   - What frustrations did they express?
   - What delighted them?

4. **Time on task:**
   - Which tasks took longest?
   - Which were completed quickly?
   - Which required hints?

## After All 3 Sessions: Prioritize Findings

### Step 1: Compile All Issues

List every issue observed across all 3 participants:

Example:
- User couldn't find Compare page (2 participants)
- Didn't understand what "xG" means (3 participants)
- Search box was hard to find (1 participant)
- Charts took long to load (2 participants)
- Loved the interactive tooltips (3 participants)

### Step 2: Count Frequency

Note how many participants experienced each issue:
- 3 participants = Critical
- 2 participants = Important
- 1 participant = Consider

### Step 3: Assess Severity

For each issue, rate severity:
- **Blocker:** Prevents task completion
- **Major:** Makes task very difficult
- **Minor:** Annoying but doesn't prevent completion
- **Cosmetic:** Visual only, doesn't affect usability

### Step 4: Apply MoSCoW Prioritization

Categorize each finding:

**MUST Fix (Critical Issues)**
- Frequency: 2-3 participants
- Severity: Blocker or Major
- Example: "Users can't find navigation menu" or "Search doesn't work"

**SHOULD Fix (Important Issues)**
- Frequency: 2-3 participants
- Severity: Minor
- OR Frequency: 1 participant, Severity: Major
- Example: "Unclear metric definitions" or "Slow loading charts"

**COULD Fix (Nice to Have)**
- Frequency: 1 participant
- Severity: Minor or Cosmetic
- Example: "Would like darker color scheme"

**WON'T Fix (Out of Scope)**
- Too complex for current timeline
- Doesn't align with project goals
- Example: "Add video highlights of players"

### Step 5: Create Prioritized List

Format:

```
MUST FIX
1. [Issue] - Observed in X/3 participants
   Problem: [Describe what happened]
   Impact: [Why it matters]
   Solution: [How to fix it]

2. [Issue] - Observed in X/3 participants
   Problem:
   Impact:
   Solution:

SHOULD FIX
[Same format]

[Same format]

[Same format with explanation]
```
